# [JVM] Volatile, Synchronize

## 자바 메모리 모델

CPU와 RAM 간의 속도 차이를 해결하기 위해 프로세서별 캐시 레이어가 존재합니다. 자바 메모리 모델은 멀티스레드 환경에서 공유 변수 접근 규칙을 정의합니다.

### 메인 메모리와 작업 메모리

![자바 메모리 모델](./img/volatile_1.png)

- **자바 스레드**: 작업의 주체
- **메인 메모리**: 모든 스레드가 접근 가능한 공간
- **작업 메모리**: 스레드가 사용하는 독립적 작업 공간

각 스레드는 메인 메모리의 변수를 복사하여 개인 작업 메모리에서 작업하며, 동기화를 통해 메인 메모리에 반영해야 합니다.

### 메모리 상호작용

읽기, 적재, 사용, 저장, 쓰기 등의 작업을 통해 동기화가 진행됩니다. 성능을 위해 필요한 순간에만 동기화되는 Lazy Evaluation 방식을 사용합니다.

## Volatile

작업 메모리를 건너뛰고 메인 메모리에 직접 값을 업데이트하는 키워드입니다.

### 캐시 무효화

"쓰기 시 즉시 메인 메모리에 기록하고, 읽기 시 최신 값을 가져옵니다." 이는 가시성을 보장하지만 원자성은 보장하지 않습니다.

### 메모리 장벽

명령어 재정렬을 방지하여 정해진 순서대로 코드 실행을 보장합니다.

## Synchronized

원자성과 가시성을 모두 보장합니다. 한 번에 하나의 스레드만 동기화 블록에 접근하며, monitorenter/monitorexit로 변환됩니다.

### 성능과 제한사항

한 스레드만 접근하므로 다른 스레드들이 대기하게 되어 성능 저하가 발생할 수 있으며, 데드락 위험도 존재합니다.
